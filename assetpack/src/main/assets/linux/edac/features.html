Introduction¶

EDAC/RAS components plugging and high-level design:

  1. Scrub control

  2. Error Check Scrub (ECS) control

  3. ACPI RAS2 features

  4. Post Package Repair (PPR) control

  5. Memory Sparing Repair control

High level design is illustrated in the following diagram:

     +-----------------------------------------------+
     |   Userspace - Rasdaemon                       |
     | +-------------+                               |
     | | RAS CXL mem |     +---------------+         |
     | |error handler|---->|               |         |
     | +-------------+     | RAS dynamic   |         |
     | +-------------+     | scrub, memory |         |
     | | RAS memory  |---->| repair control|         |
     | |error handler|     +----|----------+         |
     | +-------------+          |                    |
     +--------------------------|--------------------+
                                |
                                |
+-------------------------------|------------------------------+
|     Kernel EDAC extension for | controlling RAS Features     |
|+------------------------------|----------------------------+ |
|| EDAC Core          Sysfs EDAC| Bus                        | |
||   +--------------------------|---------------------------+| |
||   |/sys/bus/edac/devices/<dev>/scrubX/ |   | EDAC device || |
||   |/sys/bus/edac/devices/<dev>/ecsX/   |<->| EDAC MC     || |
||   |/sys/bus/edac/devices/<dev>/repairX |   | EDAC sysfs  || |
||   +---------------------------|--------------------------+| |
||                           EDAC|Bus                        | |
||                               |                           | |
||   +----------+ Get feature    |      Get feature          | |
||   |          | desc +---------|------+ desc +----------+  | |
||   |EDAC scrub|<-----| EDAC device    |      |          |  | |

||   +----------+      | feature control|      | repair   |  | |

||         ______________________|_____________              | |
|+---------|---------------|------------------|--------------+ |

|  |            |  | CXL mem driver|     | Client driver |     |
|  | ACPI RAS2  |  | scrub, ECS,   |     | memory repair |     |
|  | driver     |  | sparing, PPR  |     | features      |     |
|  +-----|------+  +-------|-------+     +------|--------+     |
|        |                 |                    |              |
+--------|-----------------|--------------------|--------------+
         |                 |                    |

|    +---|-----------------|--------------------|-------+      |
|    |                                                  |      |
|    |            Platform HW and Firmware              |      |
|    +--------------------------------------------------+      |

  1. EDAC Features components - Create feature-specific descriptors. For example: scrub, ECS, memory repair in the above diagram.

  2. EDAC device driver for controlling RAS Features - Get feature’s attribute descriptors from EDAC RAS feature component and registers device’s RAS features with EDAC bus and expose the features control attributes via sysfs. For example, /sys/bus/edac/devices/<dev-name>/<feature>X/

  3. RAS dynamic feature controller - Userspace sample modules in rasdaemon for dynamic scrub/repair control to issue scrubbing/repair when excess number of corrected memory errors are reported in a short span of time.

  1. Memory Repair
